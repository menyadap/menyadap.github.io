## Pengenalan Shortcode di Astro

Di Astro v4+, Anda dapat menggunakan **Astro Components** langsung dalam file Markdown biasa (bukan MDX) tanpa perlu import manual.

---

## 1. Menggunakan Komponen Alert

Shortcode `Alert` bisa digunakan dengan berbagai tipe:

### Alert Info

&lt;Alert type="info" title="Informasi">
Ini adalah pesan informasi penting yang ingin Anda sampaikan kepada pembaca.
&lt;/Alert>

### Alert Warning

&lt;Alert type="warning" title="Perhatian!">
Hati-hati dengan konten ini, pastikan Anda membaca dengan teliti sebelum melanjutkan.
&lt;/Alert>

### Alert Danger

&lt;Alert type="danger" title="Bahaya!">
Ini adalah peringatan serius yang harus diperhatikan dengan seksama oleh pembaca.
&lt;/Alert>

### Alert Success

&lt;Alert type="success" title="Berhasil!">
Tindakan Anda telah berhasil dilaksanakan dengan sempurna.
&lt;/Alert>

## Cara Membuat Shortcode Sendiri

Untuk membuat shortcode baru:

1. **Buat file komponen** di `src/components/NamaKomponen.astro`
2. **Tentukan props** yang diperlukan dalam interface
3. **Gunakan di Markdown** cukup dengan `<NamaKomponen prop="value" />`
4. **Tidak perlu import** - Astro secara otomatis mengenali komponen

### Contoh Struktur Komponen

```astro
---
// src/components/MyShortcode.astro
interface Props {
  title: string;
  content?: string;
}

const { title, content } = Astro.props;
---

&lt;div class="shortcode">
  <h3>{title}&lt;/h3>
  &lt;slot />
  {content && <p>{content}&lt;/p>}
&lt;/div>

<style>
  .shortcode {
    padding: 1rem;
    border: 1px solid #ccc;
  }
&lt;/style>
```

---

## Tips & Best Practices

- ✅ Gunakan **PascalCase** untuk nama komponen (contoh: `MyComponent`)
- ✅ Gunakan **slot** untuk konten dinamis
- ✅ Tentukan **props default** agar komponen fleksibel
- ✅ Tambahkan **scoped styles** dengan `<style>` tag
- ❌ Jangan gunakan JavaScript logic kompleks di Markdown
- ❌ Jangan bergantung pada library eksternal yang berat

---

## Konfigurasi Astro

Pastikan `astro.config.mjs` sudah mengaktifkan markdown dengan components:

```javascript
export default defineConfig({
  markdown: {
    shikiConfig: {
      theme: 'github-dark',
      wrap: true,
    },
  },
});
```

---

**Catatan:** Fitur ini memerlukan Astro v4.0 atau lebih tinggi dan menggunakan markdown biasa (bukan MDX).
